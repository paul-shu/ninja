00005 Ninja is a small build system closest in spirit to Make.
00006 
00007 See the manual -- http://martine.github.com/ninja/manual.html or
00008 manual.asciidoc included in the distribution -- for more background,

00017  * \section HACKING
00018  * \verbatim
00019 Adjusting build flags:
00020   CFLAGS=-O3 ./configure
00021   and rebuild.
00022 
00023 Test-driven development:
00024   Set your build command to
00025     ./ninja ninja_test && ./ninja_test --gtest_filter=MyTest.Name
00026   now you can repeatedly run that while developing until the tests pass.
00027   Remember to build "all" before committing to verify the other source
00028   still works!
00029 
00030 Coding guidelines:
00031 - Function name are camelcase.
00032 - Member methods are camelcase, expect for trivial getters which are
00033   underscore separated.
00034 - Local variables are underscore separated.
00035 - Member variables are underscore separated and suffixed by an extra underscore.
00036 - Two spaces indentation.
00037 - Opening braces is at the end of line.
00038 - Lines are 80 columns maximum.
00039 - All source files should have the Google Inc. license header.
00040 - Also follow this style:
00041   http://google-styleguide.googlecode.com/svn/trunk/cppguide.xml
00042 
00043 Documentation guidelines:
00044 - Use /// for doxygen.
00045 - Use \a to refer to arguments.
00046 - It's not necessary to document each argument, especially when they're
00047   relatively self-evident (e.g. in CanonicalizePath(string* path, string* err),
00048   the arguments are hopefully obvious)
00049 
00050 Generating the manual:
00051   sudo apt-get install asciidoc --no-install-recommends
00052   ./ninja manual
00053 
00054 Windows development on Linux (this is kind of hacky right now):
00055 - sudo apt-get install gcc-mingw32 wine
00056 - ./configure.py --platform=mingw
00057 - Build gtest:
00058   - unpack it into your source dir
00059   - ./configure CC=i586-mingw32msvc-cc CXX=i586-mingw32msvc-c++
00060   - to work around missing _TlsAlloc I had to hack the libtool script to
00061     append -lmingw32 -lkernel32 at the *end* of the link line
00062 - Build ninja: /path/to/linux/ninja ninja
00063 - Run: ./ninja  (implicitly runs through wine(!))
00064 
00065 Using clang:
00066 - Enable colors manually:
00067   CXX='/path/to/llvm/Release+Asserts/bin/clang++ -fcolor-diagnostics' ./configure.py
